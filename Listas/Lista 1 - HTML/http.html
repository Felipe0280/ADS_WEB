Protocolo HTTP

O que é o HTTP?

HTTP (HyperText Transfer Protocol) é o protocolo responsável pela comunicação entre navegadores e servidores na Web. Sempre que um usuário acessa um site, o navegador envia uma requisição ao servidor, que responde com os recursos solicitados, como páginas HTML, imagens, vídeos, etc.

Papel do HTTP na comunicação entre navegador e servidor

O HTTP estabelece regras para a troca de informações entre o cliente (navegador) e o servidor. Quando o usuário acessa um endereço na internet, o navegador envia uma requisição HTTP ao servidor. O servidor processa essa solicitação e envia uma resposta, contendo o conteúdo solicitado ou uma mensagem de erro.

Cliente, Servidor, Requisição e Resposta

Cliente

É o dispositivo ou programa que faz a solicitação ao servidor. Normalmente é um navegador como Google Chrome, Mozilla Firefox ou o finado Internet Explorer.

Servidor

É o computador responsável por armazenar o site e responder às solicitações dos clientes.

Requisição (Request)

É a mensagem enviada pelo cliente solicitando um recurso ao servidor.

Resposta (Response)

É a mensagem enviada pelo servidor contendo o recurso solicitado ou um código informando o resultado da requisição.

Métodos HTTP

GET

O método GET é utilizado para solicitar informações ao servidor. É usado para consultar páginas, pesquisar informações ou acessar recursos, sem alterar os dados existentes.

POST

O método POST é utilizado para enviar dados ao servidor. É comum em formulários do cadastro, login, envio de mensagens e upload de arquivos.

Exemplo de uma Requisição HTTP

        GET /index.html HTTP/1.1
        Host: www.exemplo.com
        User-Agent: Mozilla/5.0
        Accept: text/html

Exemplo de uma Resposta HTTP

        HTTP/1.1 200 OK
        Content-Type: text/html
        
            
                Página carregada com sucesso.

Códigos de Status HTTP

200 - OK

A requisição foi realizada com sucesso e o servidor retornou o recurso solicitado.

301 - Moved Permanently

O recurso foi movido permanentemente para outro endereço. O navegador deve utilizar o novo endereço nas próximas solicitações.

400 - Bad Request

O servidor não conseguiu entender a requisição porque ela foi enviada de forma incorreta.

403 - Forbidden

O servidor entendeu a requisição, porém o cliente não possui permissão para acessar o recurso solicitado.

404 - Not Found

O recurso solicitado não foi encontrado no servidor.

500 - Internal Server Error

O servidor encontrou um erro interno e não conseguiu concluir a requisição.
